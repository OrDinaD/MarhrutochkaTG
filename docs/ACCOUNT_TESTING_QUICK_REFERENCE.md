# 🚀 Быстрый справочник: Тестирование аккаунтов

## 📚 Для пользователя

### Как протестировать работу аккаунта?

#### Шаг 1: Добавление аккаунта
```
1. Отправьте команду /account боту
2. Нажмите "➕ Добавить аккаунт"
3. Введите номер телефона (9 цифр, без +375)
   Пример: 299605390
4. Введите пароль от сайта маршруток
5. Подождите подтверждения "✅ Аккаунт успешно добавлен!"
```

✅ **Что проверить:**
- Сообщение с паролем должно удалиться автоматически
- Вы должны увидеть подтверждение об успешном добавлении
- При повторном вызове /account должна появиться кнопка "🔄 Изменить аккаунт"

#### Шаг 2: Проверка аккаунта (через автопокупку)
```
1. Отправьте команду /autobuy
2. Следуйте инструкциям:
   - Город отправления: Минск
   - Город назначения: Островец
   - Дата: ГГГГ-ММ-ДД (например, 2025-12-01)
   - Время: 07:00 (или - для пропуска)
3. Нажмите "✅ Купить"
4. Дождитесь результата
```

⚠️ **ВНИМАНИЕ:** Это может привести к реальной покупке билета!

✅ **Что проверить:**
- Бот не должен запрашивать аккаунт снова
- Должно появиться сообщение "⏳ Идет поиск и покупка билета..."
- В конце должен быть результат (успех или ошибка)

#### Шаг 3: Тест с неверными данными
```
1. Отправьте /account
2. Нажмите "🔄 Изменить аккаунт"
3. Введите правильный номер
4. Введите неправильный пароль
5. Попробуйте /autobuy
```

✅ **Что проверить:**
- Должна быть ошибка "❌ Ошибка авторизации"
- Должно быть предложение проверить учетные данные

```bash

    --cov-report=html \
```

### Ручное тестирование

#### Подготовка
```bash
# 1. Установить зависимости
pip install -r requirements.txt
pip install -r requirements.dev.txt

# 2. Установить Playwright браузер
playwright install chromium

# 3. Настроить .env
cp .env.example .env
# Отредактировать .env:
# - TELEGRAM_BOT_TOKEN=ваш_токен
# - ADMIN_TELEGRAM_ID=ваш_id
# - ENCRYPTION_KEY=ваш_ключ

# 4. Запустить бота
python main.py
```

#### Генерация ключа шифрования
```python
from cryptography.fernet import Fernet
print(Fernet.generate_key().decode())
```

#### Проверка безопасности
```bash
# Проверить что пароли зашифрованы
cat data/user_accounts.json
# Не должно быть plaintext паролей!

# Проверить права доступа (Unix)
ls -la data/user_accounts.json
# Должно быть -rw------- или -rw-r--r--

# Проверить логи
tail -f logs/bot.log | grep -i password
# Пароли не должны попадать в логи!
```

## 🐛 Известные проблемы

### 1. Слабое шифрование (КРИТИЧНО)
**Проблема:** Используется простое XOR шифрование  
**Решение:** Переход на cryptography.Fernet  
**Статус:** 🔴 Требует исправления

### 2. Ключ по умолчанию (КРИТИЧНО)
**Проблема:** `default_key_change_in_production` в коде  
**Решение:** Обязательная переменная окружения  
**Статус:** 🔴 Требует исправления

### 3. Race conditions (СРЕДНЕ)
**Проблема:** Конкурентная запись в один файл  
**Решение:** Использовать file locking или Redis  
**Статус:** 🟡 Низкая вероятность, но возможно

### 4. Отсутствие ротации ключей (НИЗКО)
**Проблема:** Нет механизма обновления ключа  
**Решение:** Добавить migration script  
**Статус:** 🟢 Nice-to-have
