## 🔍 Ручное тестирование в Telegram

### 1. Базовые команды:
- `/start` - должно показать главное меню
- `/help` - справка по использованию  
- `/monitoring` - управление мониторингом

### 2. Поиск рейсов:
- Отправьте `2025-07-02` - должен показать рейсы на завтра
- Используйте кнопку "🔍 Поиск рейсов" и выберите дату

### 3. Настройка мониторинга (пошагово):

**Шаг 1: Запуск**
- Нажмите "🔔 Настроить мониторинг"
- Должен появиться выбор даты

**Шаг 2: Дата**
- Выберите "🔵 Сегодня" или "🟢 Завтра"
- Или нажмите "📅 Другая дата" и введите `2025-07-03`

**Шаг 3: Направление**
- Выберите "🏙️ Минск → Островец" или "🏘️ Островец → Минск"
- Или "🔄 Оба направления"

**Шаг 4: Тип времени**
- "🚀 Время отправления" - если важно когда уехать
- "🎯 Время прибытия" - если важно когда приехать
- "⏰ Любое время" - любые рейсы

**Шаг 5: Диапазон времени** (если не выбрано "любое время")
- "🌅 Утром (05:00-09:00)"
- "☀️ Днём (09:00-15:00)"
- "🌆 Вечером (15:00-20:00)"
- "🌙 Ночью (20:00-05:00)"
- "🕐 Пользовательский диапазон" - введите `07:30-09:30`

**Шаг 6: Подтверждение**
- Проверьте настройки
- Нажмите "✅ Да, запустить!"

### 4. Управление мониторингом:
- `/monitoring` - посмотреть активный мониторинг
- "🛑 Остановить" - остановить мониторинг
- "📱 Проверить сейчас" - ручная проверка
- "🔧 Изменить" - новые настройки

### 5. Проверка уведомлений:
- Настройте мониторинг на ближайшее время
- Подождите до 5 минут
- Должно прийти уведомление при наличии мест

## 🧪 Автоматическое тестирование

Для проверки работоспособности всех ключевых компонентов и анализа покрытия кода тестами, используйте следующую команду из корневой директории проекта:

```bash
# Убедитесь, что виртуальное окружение активировано
source venv/bin/activate

# Запуск всех тестов с генерацией отчета о покрытии
coverage run --source=src -m unittest discover tests && coverage report -m
```

Эта команда:
1. Запустит все тесты, находящиеся в директории `tests/`.
2. Соберет данные о том, какие строки исходного кода в `src/` были выполнены в ходе тестов.
3. Выведет в консоль детальный отчет, показывающий процент покрытия для каждого модуля.

## ❗ Troubleshooting:

**Бот не отвечает:**
- Проверьте, что процесс запущен
- Убедитесь, что токен корректный
- Попробуйте перезапустить: `./start_bot.sh`

**Не приходят уведомления:**
- Проверьте, что мониторинг активен (`/monitoring`)
- Возможно, нет подходящих рейсов по критериям
- Попробуйте "📱 Проверить сейчас"

**Ошибки парсера:**
- Сайт может быть временно недоступен
- Проверьте интернет-соединение
- Запустите `python full_test.py` для диагностики
