# 🚂 Деплой MarhrutochkaTG на Railway

## Подготовка к деплою

### 1. Создание аккаунта на Railway
1. Зарегистрируйтесь на [Railway.app](https://railway.app/)
2. Привяжите свой GitHub-аккаунт

### 2. Настройка проекта
1. Войдите в Railway Dashboard
2. Нажмите "New Project" → "Deploy from GitHub repo"
3. Выберите репозиторий `OrDinaD/MarhrutochkaTG`

## Конфигурация проекта

### 3. Настройка переменных окружения
В Railway Dashboard перейдите в раздел **Variables** и добавьте следующие переменные:

```bash
# Telegram Bot Token (обязательно)
TELEGRAM_BOT_TOKEN=YOUR_BOT_TOKEN_HERE

# Настройки логирования (опционально)
LOG_LEVEL=INFO

# Настройки мониторинга (опционально)
MONITOR_INTERVAL=300  # 5 минут в секундах
MAX_MONITORS_PER_USER=3

# Настройки парсера (опционально)
REQUEST_TIMEOUT=30
MAX_RETRIES=3
```

### 4. Конфигурация сборки и запуска

Railway автоматически определит Python-проект, но для гарантии создайте файл `railway.toml`:

```toml
[build]
builder = "NIXPACKS"

[deploy]
healthcheckPath = "/"
healthcheckTimeout = 300
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 10
```

### 5. Настройка Procfile (опционально)
Если Railway не определит команду запуска автоматически, создайте `Procfile`:

```
web: python src/bot.py
```

## Проверка развертывания

### 6. Файлы для проверки
Убедитесь, что в репозитории есть следующие файлы:

- ✅ `requirements.txt` - зависимости Python
- ✅ `src/bot.py` - основной файл бота
- ✅ `src/parser.py` - парсер сайта
- ✅ `src/requests_auth.py` - менеджер аутентификации
- ✅ `README.md` - документация

### 7. Проверка requirements.txt
Убедитесь, что файл содержит все нужные зависимости:

```
python-telegram-bot>=21.0.0
requests>=2.31.0
beautifulsoup4>=4.12.0
lxml>=4.9.0
python-dotenv>=1.0.0
apscheduler>=3.10.0
```

## Деплой и запуск

### 8. Автоматический деплой
1. После подключения репозитория Railway начнет автоматическую сборку
2. Следите за логами в разделе **Deployments**
3. Процесс сборки может занять 2-5 минут

### 9. Проверка логов
После деплоя проверьте логи в разделе **Logs**:

```
✅ Успешный запуск:
INFO:__main__:Бот запущен успешно
INFO:telegram.ext.Application:Application started

❌ Ошибки:
ERROR:telegram.error.InvalidToken: Invalid token
ERROR:ModuleNotFoundError: No module named 'module_name'
```

### 10. Тестирование бота
1. Найдите вашего бота в Telegram
2. Отправьте команду `/start`
3. Проверьте основные функции:
   - Поиск рейсов
   - Настройка мониторинга
   - Вход в аккаунт

## Мониторинг и управление

### 11. Настройка автоматического масштабирования
В разделе **Settings** → **Deploy**:

```
Memory: 512 MB (рекомендуется)
CPU: 0.5 vCPU (достаточно для начала)
Replicas: 1 (один экземпляр)
```

### 12. Настройка доменов (опционально)
В разделе **Settings** → **Domains**:
- Railway автоматически создаст поддомен
- Можно добавить кастомный домен

### 13. Мониторинг метрик
В разделе **Metrics** следите за:
- Использованием CPU
- Потреблением памяти
- Количеством запросов
- Временем отклика

## Возможные проблемы и решения

### 14. Частые ошибки

**Ошибка: "Application failed to start"**
```bash
# Проверьте логи и убедитесь, что:
1. TELEGRAM_BOT_TOKEN задан правильно
2. requirements.txt содержит все зависимости
3. src/bot.py существует и запускается
```

**Ошибка: "ModuleNotFoundError"**
```bash
# Добавьте отсутствующую зависимость в requirements.txt
# Пример: requests>=2.31.0
```

**Ошибка: "Memory limit exceeded"**
```bash
# Увеличьте лимит памяти в настройках до 1GB
```

**Ошибка: "Timeout"**
```bash
# Увеличьте timeout в railway.toml:
[deploy]
healthcheckTimeout = 600
```

### 15. Отладка
Для отладки используйте:
1. Логи Railway в реальном времени
2. Команду `/start` для тестирования бота
3. Проверку переменных окружения

## Обновления

### 16. Автоматические обновления
Railway автоматически разворачивает изменения при:
- Push в основную ветку GitHub
- Изменении переменных окружения
- Ручном перезапуске через Dashboard

### 17. Откат изменений
В разделе **Deployments** можно:
- Откатиться к предыдущей версии
- Просмотреть историю деплоев
- Сравнить изменения

## Полезные команды

### 18. CLI Railway (опционально)
Установите Railway CLI для управления из терминала:

```bash
# Установка
npm install -g @railway/cli

# Логин
railway login

# Деплой
railway up

# Просмотр логов
railway logs

# Переменные окружения
railway variables
```
