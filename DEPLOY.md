# 🚀 Деплой бота на Railway

## 📋 Подготовка к деплою

### 1. Установка Railway CLI
Для деплоя на Railway необходимо установить Railway CLI:

```bash
npm i -g @railway/cli
```

### 2. Авторизация в Railway
```bash
railway login
```

### 3. Подготовка переменных окружения
Для работы бота необходимы следующие переменные окружения:

- `TELEGRAM_BOT_TOKEN` - токен основного бота
- `LOG_BOT_TOKEN` - токен бота для логирования (можно создать отдельного бота)
- `LOG_CHAT_ID` - ID чата для отправки логов (можно использовать личный чат с ботом)

## 🚀 Деплой

### Автоматический деплой
Используйте скрипт для автоматического деплоя:
```bash
./deploy_to_railway.sh
```

### Ручной деплой
1. Привяжите проект к Railway
```bash
railway link
```

2. Настройте переменные окружения:
```bash
railway variables set TELEGRAM_BOT_TOKEN=ваш_токен
railway variables set LOG_BOT_TOKEN=токен_бота_для_логов
railway variables set LOG_CHAT_ID=ид_чата_для_логов
railway variables set PYTHONUNBUFFERED=1
railway variables set TZ=Europe/Moscow
```

3. Выполните деплой:
```bash
railway up
```

## 📊 Мониторинг логов

### Через бота
Бот отправляет логи в указанный чат (LOG_CHAT_ID) с помощью отдельного бота (LOG_BOT_TOKEN).
- 🔴 Ошибки отправляются мгновенно
- 🟠 Предупреждения собираются в буфер
- 🟢 Информационные сообщения фильтруются

### Через CLI
Используйте скрипт для просмотра логов:
```bash
./monitor_logs.sh
```

Или напрямую через Railway CLI:
```bash
railway logs
```

## ⚙️ Управление ботом

Используйте скрипт для управления ботом:
```bash
./manage_bot.sh
```

Возможности скрипта:
- Просмотр логов в реальном времени
- Просмотр статуса деплоя
- Перезапуск бота
- Обновление переменных окружения
- Обновление кода (новый деплой)

### Ручные команды

**Перезапуск бота:**
```bash
railway service restart
```

**Просмотр статуса:**
```bash
railway status
```

**Просмотр логов в реальном времени:**
```bash
railway logs -f
```

## 🚑 Устранение неполадок

1. Проверьте логи бота:
   - В Telegram-чате
   - Через Railway CLI

2. Проверьте статус деплоя:
   ```bash
   railway status
   ```

3. Перезапустите бота:
   ```bash
   railway service restart
   ```

4. Проверьте переменные окружения:
   ```bash
   railway variables list
   ```
